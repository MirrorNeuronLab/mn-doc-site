# MirrorNeuron Documentation

MirrorNeuron is a local-first runtime for durable, message-driven AI workflows. It can run on one machine or across a small cluster, and it provides a CLI, REST API, local model support, and a blueprint runtime.

## Start Here

1. [Installation](installation.md): install the local runtime and its dependencies.
2. [Quickstart](quickstart.md): validate and submit a checked-in blueprint.
3. [Examples](examples.md): choose a blueprint that matches the workflow you want to explore.
4. [Security Model](security.md): understand the trust boundaries before running third-party or production workloads.

## Safe First Run

Before you run a workflow, inspect its `manifest.json`, payloads, model requirements, and `pass_env` entries. Then validate it locally:

```bash
mn blueprint validate otterdesk-blueprints/generic_customer_service_voice_coworker
```

Validation does not submit a job. When the blueprint's required services and inputs are ready, start the runtime and run it:

```bash
mn runtime start
mn blueprint run --folder otterdesk-blueprints/generic_customer_service_voice_coworker
```

Use [Quickstart](quickstart.md) for the full flow and [Security Model](security.md) before exposing a runtime beyond your trusted local network.
